Develop-12|지정한 시간에 '실행하는, setTimeout 코어 메서드

.js로 시작하는 '파일'은 실행주체와의 규약으로, 파일 전체를 한번 훑어내려가며 실행합니다. 따라서 우리가 html 방식으로 문서를 실행하는 일과, Node.js로 명령을 통해 실행하는 일 모두 '일괄 실행'이 에러가 없다면 모두 진행됩니다.(되도록 에러를 안만들고 실행 해주는 것이 자바스크립트의 매우 큰 단점입니다) 파일 내에서도, 주요 부분에서만 실행하도록 '호출'이라는 개념을 통해 원하는 만큼 실행시점을 조정, 제어할 수 있는 함수는 파일이라는 큰 틀 속에 실행 주체중 하나가 된다고 볼 수 있습니다.

사용자입장에서는, 웹페이지 접속(패킷통신을 생각하면, 접속보다는 다운로드가 알맞은 표현입니다)이 대표적인 '파일을 읽는 행위'가 될 것이고, 어떤 요소를 조작하는 이벤트 발생이 '어떤 함수를 실행하는 행위'가 되어줄 것입니다. 일반적이고 비교적 익숙한 방식이라고 할 수 있습니다. 사용자나, 개발자가 주체가 되는 일과 약간 다르게,'정해진 시간',혹은 '원하는 시간만큼 이후'라는 시간개념도 프로그래밍에서 상당히 중요한 이벤트 트리거가 됩니다. 아침에 잠에서 깨게끔 도와주는 기상 알람과 비슷합니다.

필요한 시간에, 적절하게 알람을 울려주는 일이되는데 에서는 상당히 강력한 메서드를 지원합니다. 날짜 관념의 Date객체는 '정보'성격에 해당하고, 본 예제가 말하는 'setTimeout'은, 실행된뒤 '얼마 후 한번 실행'을 이야기합니다. 상당히 절묘하며, 복합적으로 활용하는 대표적인 메인 메서드 중에 하나입니다. 이해를 돕기 위해 안티패턴이긴하지만, 가해성을 고려해 예제를 구성하였습니다.
